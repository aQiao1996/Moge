# 任务完成标准与AI响应

## 18. 任务完成标准

每个任务完成前必须检查以下清单：

- [ ] **Lint检查**：`pnpm run lint` 检查通过，无警告无错误
- [ ] **类型检查**：`pnpm run typecheck` 类型检查通过
- [ ] **功能测试**：功能手动测试通过，包括正常流程和异常情况
- [ ] **文档更新**：相关注释和文档已更新
- [ ] **代码注释**：代码注释完整，复杂逻辑有中文说明
- [ ] **调试清理**：无调试用console.log遗留（除必要的error日志）
- [ ] **三端对应**：三端字段对应检查通过（如涉及数据库）
- [ ] **确认提示**：危险操作有确认提示（如涉及删除/清空）

## 19. AI响应规范

- 🎯 **明确方案**：提供多个方案时说明各自优缺点、适用场景、实现难度
- 📊 **影响评估**：清晰说明改动会影响哪些文件、功能、接口
- 🔍 **问题定位**：先定位问题根本原因，再提出解决方案，避免治标不治本
- 💬 **持续反馈**：长时间任务提供阶段性进度反馈，让用户了解当前状态
- ❌ **失败处理**：遇到错误时说明原因、已尝试的方法、替代方案
- 📋 **方案对比**：重要决策时列出不同方案的对比表格，帮助用户选择
- 🤔 **主动询问**：需求不明确时主动询问，避免基于错误假设开发
