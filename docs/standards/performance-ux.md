# 性能优化与用户体验

## 15. 性能优化意识

- 🚀 **查询优化**：避免在循环中查询数据库，使用批量查询或预加载
- 💾 **缓存策略**：合理使用缓存减少重复计算和数据库访问
- 📦 **懒加载**：大列表使用分页或虚拟滚动，避免一次性加载过多数据
- 🎨 **组件优化**：React组件合理使用`React.memo`、`useMemo`、`useCallback`避免不必要的重渲染
- 📊 **数据量意识**：考虑数据量增长场景，避免算法复杂度过高

## 16. 用户体验规范

- ⏳ **加载状态**：所有异步操作必须显示loading状态，避免用户误以为界面卡死
- ❓ **确认提示**：删除、清空、重置等危险操作需要二次确认对话框
- ♿ **可访问性**：表单元素有proper label，按钮有描述性文本，支持键盘操作
- 📱 **响应式**：重要页面考虑移动端显示效果，使用响应式布局
- 💬 **友好提示**：错误提示要具体明确，说明问题原因和解决方法

## 17. 代码完整性

- 🔄 **完整流程**：CRUD功能要完整实现（创建、读取、更新、删除），不留半成品
- 🎯 **一致性**：同类功能保持相同的实现模式和代码风格
- 🧹 **清理工作**：创建文件后确保在合适位置export和import，不留孤立文件
- 📚 **依赖完整**：新增功能确保所有依赖都已安装，版本在catalog中定义
- 🔗 **关联更新**：修改接口/类型定义后，同步更新所有相关的前端/后端代码
